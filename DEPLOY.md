# Deployment Guide

This project is deployed with `OpenNext + Cloudflare Workers`.

## Prerequisites

- Node.js and npm
- A Cloudflare account
- `wrangler` authentication
- A target domain for `NEXT_PUBLIC_SITE_URL`

## First Deployment

### 1. Install dependencies

```bash
npm install
cp .env.example .env.local
```

At minimum, prepare:

```env
NEXT_PUBLIC_SITE_URL=https://your-domain.com
ADMIN_PASSWORD=change-me
ADMIN_TOKEN_SALT=replace-with-a-random-secret
AI_CONFIG_ENCRYPTION_SECRET=replace-with-another-random-secret
```

### 2. Log in to Cloudflare

```bash
npx wrangler login
```

### 3. Provision bindings

```bash
npm run cf:init -- --site-url=https://your-domain.com
```

This generates `wrangler.local.toml` with concrete resource bindings.

Optional:

- Add `--with-kv` if you also want KV-based public cache support.
- Add a private `SKILLS` bucket if you enable editor Skills uploads.

### 4. Set secrets

```bash
npx wrangler secret put ADMIN_PASSWORD -c wrangler.local.toml
npx wrangler secret put ADMIN_TOKEN_SALT -c wrangler.local.toml
npx wrangler secret put AI_CONFIG_ENCRYPTION_SECRET -c wrangler.local.toml
```

Optional AI secret:

```bash
npx wrangler secret put AI_API_KEY -c wrangler.local.toml
```

Optional Workers AI discovery credentials:

```bash
npx wrangler secret put CLOUDFLARE_ACCOUNT_ID -c wrangler.local.toml
npx wrangler secret put CLOUDFLARE_API_TOKEN -c wrangler.local.toml
```

### 5. Build and deploy

```bash
npm run cf-typegen
npm run build
npm run deploy
```

## Local Worker Preview

```bash
npm run preview
```

The preview flow prefers `wrangler.local.toml`. The template `wrangler.toml` is intentionally generic and should not be treated as a production-ready resource manifest by itself.

## Ongoing Updates

```bash
git pull
npm install
npm run verify
npm run deploy
```

## Required Bindings

- `DB` → `D1`
- `IMAGES` → public `R2`
- `SKILLS` → private `R2` if Skills are enabled
- `ASSETS` → generated by OpenNext build output

Optional bindings:

- `WORKERS_AI`
- queue bindings for background jobs
- KV cache bindings

## Common Problems

### Missing `D1` or `R2`

Run:

```bash
npm run cf:init -- --site-url=https://your-domain.com
```

### Admin login says authentication is incomplete

Set at least:

```bash
npx wrangler secret put ADMIN_PASSWORD -c wrangler.local.toml
npx wrangler secret put ADMIN_TOKEN_SALT -c wrangler.local.toml
```

### Saved AI provider keys can no longer be decrypted

Usually `AI_CONFIG_ENCRYPTION_SECRET` or `ADMIN_TOKEN_SALT` changed after the values were saved. Keep both stable across deployments.

### Canonical URL / sitemap / RSS point to the wrong domain

Check:

- `.env.local`
- `wrangler.local.toml`
- Cloudflare runtime vars / secrets

`NEXT_PUBLIC_SITE_URL` must be consistent everywhere.

### `WORKER_SELF_REFERENCE` not found during deploy

Make sure the deployed Worker script name and the service binding target are identical. In practice:

- `wrangler.toml` `name`
- `[[services]].service`
- Cloudflare Build / Deploy target name

must all resolve to the same Worker script name.
